# 开发任务编排层近期收尾方案

更新时间：2026-06-23

状态：已落地并提交。

最近提交：`8f7a60d 产品化开发任务状态入口`

## Summary

本轮没有进入 P2-A `requirement_id` / `trace_links`，而是先把已经落地的 P0/P1 dev task 后端能力在前端产品化。用户现在可以在当前会话里看到任务状态条，也可以通过聊天工具栏的“任务”入口打开轻量任务抽屉，查看当前 session 或全部任务。

实现保持轻量工具型风格：不新增复杂任务管理页，不改变 dev task 后端推进语义，不新增本地关键词任务路由。

## 已完成

### 前端类型与 API

- [x] 增加 `PersonalDevTask`、`PersonalDevTaskStage`、`PersonalDevTaskNextAction`、`PersonalDevTaskValidationSummary`。
- [x] 增加 `devTaskList(sessionUid?: string, status?: string)`、`devTaskGet(taskUid: string)`、`devTaskContinue(taskUid: string)`。
- [x] `draftList` 支持可选 `task_uid`。
- [x] 后端 `GET /api/personal/drafts` 支持 `task_uid`；同时传 `session_uid` 和 `task_uid` 时取交集。

### 当前任务轻量展示

- [x] 主聊天消息流上方增加“当前任务”紧凑状态条。
- [x] 当前任务查询从只查 active/blocked 改为当前 session 全量择优，避免 completed/archived 后完全不可见。
- [x] 展示任务标题、状态、当前/下一阶段、阶段进度、blocked reason、最近动作、验证摘要。
- [x] 阶段进度使用小型 tags 表达 `done` / `pending` / `needs_revision`。
- [x] `needs_revision` 和 `blocked` 使用克制状态色。
- [x] “继续推进”仍通过 `chatTurn({ session_uid, content: "继续" })` 进入对话历史，没有绕过聊天。

### 任务入口

- [x] 聊天工具栏新增“任务”按钮。
- [x] 任务抽屉支持“当前 session / 全部任务”过滤。
- [x] 无任务时显示明确空态，避免页面上完全找不到 tasks。
- [x] 任务列表展示 task 短号、状态、阶段进度和阻塞原因。

### 草稿箱联动

- [x] 草稿列表项显示所属 task_uid 短号和文档阶段。
- [x] 当前任务状态条中的 stage 可点击打开草稿箱并定位对应 draft。
- [x] 草稿箱支持全部 / 当前 session / 当前 task 过滤。
- [x] 当前 task 过滤只请求该 task 的 drafts，不混入其它 session/task 材料。

### 运行态诊断收敛

- [x] 聊天气泡主文案保持简短。
- [x] 对 `dev_task.last_action.policy`、LLM fallback、validation summary、tool result error 增加可折叠“诊断”区域。
- [x] 诊断只展示 provider/model/call_id/error summary/policy reason。
- [x] task 状态区显示验证分类：`passed`、`code_logic`、`test_expectation`、`timeout`、`config`、`unknown`。

### 发送生命周期修复

- [x] `apiPost` / `chatTurn` 支持 `AbortSignal`。
- [x] 发送中按钮变为“停止本次回复”。
- [x] optimistic pending 和 local error 按 session 隔离，切换会话不再导致 pending 气泡消失但按钮仍转圈。
- [x] 取消请求后恢复输入草稿，并清理对应 session 的 pending 状态。
